# GitX 使用说明

## 一、部署安装

### 1. 编译打包（开发者）
```bash
cd GitX
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true
```

输出目录：`./bin/Release/net8.0-windows/win-x64/publish/GitX.exe`

### 2. 团队分发
将以下文件分发给团队成员：
- `GitX.exe` — 主程序（绿色免安装，内置 .NET 8 运行时）
- `RegisterGitXMenu.reg` — 右键菜单注册脚本

### 3. 注册右键菜单
1. 右键编辑 `RegisterGitXMenu.reg`
2. 将 `"C:\\Tools\\GitX\\GitX.exe"` 替换为实际安装路径
3. 双击运行 `RegisterGitXMenu.reg`
4. 确认导入注册表

## 二、使用流程

### 1. 启动
在任意 Git 仓库目录右键 → **GitX - 和其他分支比较**

程序自动启动并传入当前仓库路径。

### 2. 选择分支
- 程序加载所有本地分支和远程分支
- 搜索框实时过滤分支名称
- 双击或选中分支后点击「确认对比」

### 3. 查看差异
- **左侧目录树**：仅显示存在变更的文件和文件夹，无差异文件自动隐藏
- **文件类型标记**：
  - 🟢 新增文件
  - 🟡 修改文件
  - 🔴 删除文件
- **右侧双栏**：
  - 上栏 = 目标分支文件内容
  - 下栏 = 本地当前分支工作区内容
  - 双编辑器滚动同步
  - 自动语法高亮（Java/C#/JS/Vue/SQL/TXT 等）
- **二进制文件**：提示「二进制文件，仅可覆盖，无法查看差异」

### 4. 覆盖文件
- **右键单个文件** → 覆盖此文件
- **右键文件夹** → 覆盖整个文件夹
- **顶部按钮** → 全部下载（覆盖所有差异文件）
- 每次覆盖前弹窗确认，避免误操作

覆盖逻辑：直接将目标分支文件内容写入本地物理文件，不执行 git merge/commit，实现选择性合并。

### 5. 关闭
关闭程序自动清理内存缓存和临时文件，不残留数据。

## 三、运行依赖

- Windows 10/11 x64
- Git for Windows（已安装）
- 无需安装 .NET 运行时（已内置）

## 四、日志与数据库

- 操作日志保存在：`%AppData%\GitX\log.db`（SQLite）
- 记录：分支对比记录、文件覆盖记录
- 崩溃日志导出到桌面：`GitX_CrashLog.txt`

## 五、常见问题

**Q: 程序启动时提示「请从 TortoiseGit 右键菜单启动」**
A: 请确保通过右键菜单启动，或手动传入 `--repo=仓库路径` 参数。

**Q: 远程分支列表不显示最新分支**
A: 程序启动时自动 Fetch 远程，若网络不通则使用本地缓存。手动点击刷新按钮可重新同步。

**Q: 覆盖文件失败「权限不足」**
A: 文件可能被其他程序占用或权限不足。关闭占用程序后重试，或手动修改文件权限。

**Q: 大仓库加载卡顿**
A: 差异计算在后台线程执行，UI 不会冻结。超过 1000 个文件时加载时间可能稍长，请耐心等待。

## 六、卸载

1. 删除 `GitX.exe` 所在文件夹
2. 运行 `regedit` 删除 `HKEY_CURRENT_USER\Software\TortoiseGit\ContextMenuEntries\GitXCompareBranch`
3. 删除 `%AppData%\GitX\` 日志目录
