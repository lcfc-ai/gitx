- 启动 → 选分支 → 打开主窗口：不拉取
- 工具栏「刷新」：只重算本地差异，不拉取
- 工具栏「拉取最新」：唯一会触发 fetch 的入口

### 改动
- `BranchSelectViewModel.LoadBranchesAsync` 删 `FetchRemote()`
- `MainWindowViewModel.RefreshDiffAsync` 删 `RunFetchAsync()`

### 设计原则
- **网络/IO 操作不要放在初始化路径或高频操作里**，必须显式由用户触发
- 涉及远端通信的功能，UI 入口必须独立于「刷新」「加载」之类的本地操作

## 性能优化（系统排查后全量修复）

### 1. LCS → Myers diff（TextDiffService）
- 原 `BuildLcsTable` 分配 `(N+1)*(M+1)` 二维 int 数组
- 10000 行文件 = 1 亿 int = ~400MB 内存，连续调用两次（行+token）会反复触发 GC
- 改为 Myers diff：O((N+M)*D) 时间复杂度，O(N+M) 内存
- 配套优化：
  - 行/Token 级 Myers 共享同一反推逻辑
  - Token 数 >600 或长度比 >3 时跳过词级 diff，整段高亮
  - 两端完全相同直接返回空 spans
  - `StringBuilder` 拼显示文本，去掉 LINQ + `string.Join` 中间数组
  - `TrimContext` 单次扫描标记保留位，去掉 LINQ 多轮枚举

### 2. EnumerateLocalFiles → git ls-files
- 原 `Directory.EnumerateFiles(AllDirectories)` 会扫 `.git/`、`node_modules/` 等大目录
- 改用 `git ls-files -z --others --exclude-standard`：单次 git 调用同时拿 tracked + untracked
- 大仓库（10万+ 文件）从 1-2 秒降到 100-200ms
- 仍保留 `EnumerateLocalFiles` 兜底（无 git.exe 时降级）
- 在 `AddFilesystemOnlyChanges` 里加路径去重 + 忽略规则检查

### 3. 拆分 _repoLock 粒度（GitService）
- 原 `GetTreeChanges` 整段加锁，包括 `NormalizeWorkingTreeChanges`（会调 `GetFileBlobInternal`）
- 拆分：先持锁拿 commit，锁外做 `Normalize`（内部子调用各自加锁）
- 同一对象的 `Diff.Compare<TreeChanges>` 与 `RetrieveStatus` 仍合在一次锁内，保证一致性
- 未跟踪列表的二次循环（`WorkingFileMatchesTarget`）在锁外，可与其他读操作并发

### 4. BuildDiffTree 合并 WorkingTreeSummary
- 原 `MainWindowViewModel.LoadDiffAsync` 调一次 `BuildDiffTree` + 一次 `GetWorkingTreeSummary`
- 后者内部又调 `GetTreeChanges`，等于跑两遍 `git diff`
- 改 `BuildDiffTreeWithSummary` 一次返回 `(root, summary)`，节省一半 git diff 耗时

### 5. LoadFileDiffAsync 缓存命中短路
- 缓存命中时直接同步赋值 `UnifiedDiffText/Rows`，跳过 `Task.Run` 和 `_fileLoadVersion` 竞争
- 关键：缓存命中是高频路径（同一文件重复点击、切回已读文件），不能拖入异步开销

### 6. TreeFilter 防抖 + FilterNode 累加
- `OnTreeFilterTextChanged` 250ms 防抖，避免每字重建整棵 ObservableCollection
- `FilterNode` 改为累加子节点 counts（DiffRoot 已有正确值，不必重算）
- `FilterWithCount` 在过滤过程中累加 visible count，省掉调用方 `LINQ Sum`

## VS 设计器体验修复

### 问题
- 用户反馈：VS 设计器里 MainWindow 看着乱，叠加态很多，无法看清实际布局

### 根因
- `d:DataContext="{d:DesignInstance Type=vm:MainWindowViewModel}"` 依赖 VM 有无参构造；
  真实 VM 构造需要 GitService/DiffService 等，VS 无法实例化
- 多个 `Visibility="{Binding IsXxx, Converter=BoolToVisibility}"` 在设计时全显示：
  加载遮罩、二进制提示、暂无提交记录、正在加载、点击左侧提交查看详情
- 没有 `DiffTreeRoot` / `PathHistoryItems` 的样例数据，列表空白

### 修复
1. 新建 `DesignTime/MainWindowViewModelSampleData.xaml`（WPF `d:DesignData` 格式），
   提供真实样例：3 个提交记录、含文件夹/文件的树、文件名/状态/diff 文案
2. `MainWindow.xaml` 改用 `d:DataContext="{d:DesignData Source=/DesignTime/...}"`
3. 给 5 个运行时遮罩加 `d:Visibility="Collapsed"`（仅设计时折叠，运行时不变）
4. `GitX.csproj` 把 `DesignTime/**/*.xaml` 从 `<Page>` 移出、放进 `<None>`，
   不进 obj/bin，运行时 0 开销

### 经验
- WPF 设计器数据：优先 `d:DesignData` + XAML 样例文件，而不是 `d:DesignInstance`
  （后者必须有无参构造且不能跑昂贵初始化）
- 「运行时遮罩」一定要配 `d:Visibility="Collapsed"`，否则设计时一片混乱
